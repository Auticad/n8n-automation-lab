1. Panoramica

Il workflow legge da un Google Sheet i prodotti in promozione, seleziona quello con la maggiore quantità disponibile e genera automaticamente — in parallelo su tre branch AI — un post social ottimizzato per Instagram/LinkedIn, un prompt immagine per generatori come Midjourney e FLUX, e un prompt video per Google Veo o Runway.

I tre output vengono ricombinati, salvati come Google Docs, convertiti in PDF con nome data-based e inviati via email come allegato. L'intera pipeline richiede un solo click e produce un pacchetto marketing completo pronto all'uso.

Tecnologie principali: Google Sheets (lettura dati prodotto) → gpt-4o-search-preview via HTTP (analisi mercato in tempo reale con web search) → tre istanze gpt-4o in parallelo via LLM Chain (copywriting social, image prompt, video prompt) → Google Drive (creazione Docs + export PDF) → Gmail (invio allegato).

Il prompt di sistema definisce un "Senior Marketing Strategist". Il prompt utente chiede un'analisi del prodotto {{ $json.Nome }} che includa:

4. Prerequisiti
n8n Istanza locale (npx n8n) o su cloud. Versione ≥ 1.x consigliata per i nodi LangChain.
Account Google Per Google Sheets, Google Drive e Gmail. Serve un OAuth2 app configurata nella Google Cloud Console.
OpenAI API Key Due credenziali distinte: una HTTP Header Auth per la chiamata a gpt-4o-search-preview, una OpenAI API nativa per i tre LLM Chain.
Google Sheet strutturato Un foglio con almeno le colonne Nome, Quantità, Promozione. I valori nella colonna Promozione devono essere esattamente "si" (lowercase).
5. Configurazione prima dell'uso
  1. Importa marketing-generator.json in n8n (Workflow → Import from file)
  2. In n8n, crea una credenziale HTTP Header Auth:
    — Nome parametro: Authorization
    — Valore: Bearer sk-... (la tua chiave OpenAI)
    Collega questa credenziale al nodo HTTP Request a OpenAI Search
  3. Crea una credenziale OpenAI API nativa con la stessa chiave API. Collegala ai tre nodi OpenAI Chat Model, OpenAI Chat Model1, OpenAI Chat Model2
  4. Crea una credenziale Google OAuth2 e collegala ai nodi Google Sheets, Google Drive (entrambi) e Gmail
  5. Apri il nodo Get row(s) in sheet e sostituisci l'ID del foglio con il tuo. Verifica che il nome del foglio sia "Articoli sportivi" o aggiornalo
  6. Apri il nodo Send a message e imposta l'indirizzo email di destinazione
  7. Attiva il workflow → click su Execute workflow per il primo test
6. Concetti chiave appresi

Fork e parallelismo: un singolo nodo può avere più connessioni in uscita verso nodi diversi. n8n esegue tutti i branch in parallelo, non in serie. Questo dimezza il tempo di esecuzione rispetto a tre chiamate LLM sequenziali.

Modelli con web search: gpt-4o-search-preview non è disponibile tramite il nodo OpenAI nativo di n8n, che supporta solo i modelli standard. Per usarlo occorre una chiamata HTTP diretta all'API con autenticazione header. La risposta ha struttura choices[0].message.content.

Sub-node LLM in n8n: i nodi chainLlm richiedono un language model provider collegato come sub-node tramite connessione di tipo ai_languageModel. Non è una connessione main standard — viene disegnata come filo tratteggiato nell'editor.

Merge by position: funziona correttamente solo se tutti gli input hanno lo stesso numero di item. Il nodo Limit prima del fork garantisce 1 item per branch, rendendo il merge deterministico.

Dati binari e Google Drive: il nodo Google Drive in modalità download restituisce un dato binario, non un JSON. Il nodo Code successivo manipola i metadati del binario ($input.item.binary.data.fileName) senza toccare il contenuto del file.
