1 Panoramica

L'utente carica una fotografia e inserisce un prompt testuale tramite un form web. Il workflow estrae i dati binari dell'immagine, li invia all'API Gemini che genera una nuova immagine elaborata, converte la risposta in file binario e la salva automaticamente su Google Drive in una cartella dedicata.

3 HTTP Request AI HTTP Request Chiama la Gemini API inviando immagine + prompt, riceve l'immagine generata
4 Convert to File Conversione Convert to File Estrae la stringa base64 dalla risposta Gemini e la riconverte in file binario PNG

Come funziona il Form Trigger n8n espone un URL pubblico (webhook) che rende accessibile il form HTML. Ogni volta che l'utente compila e invia il form, il workflow si avvia automaticamente. Il file caricato viene ricevuto come dato binario e i campi testo come JSON.

JSON Body inviato all'API:

    "parts": [

      {
        "inline_data": {
          "mime_type": "{{ $('On form submission').item.json['Carica immagine'][0].mimetype }}",

        }
      }

  "generationConfig": { "responseModalities": ["IMAGE", "TEXT"] }
}
Struttura della risposta Gemini La risposta JSON contiene candidates[0].content.parts: un array con almeno due elementi — parts[0] contiene il testo generato, parts[1].inlineData.data contiene l'immagine generata come stringa base64.

Operazione Create vs Update Usa sempre Create (o Upload) per salvare un nuovo file in una cartella — passando l'ID della cartella come destinazione. L'operazione Update serve invece per sovrascrivere un file già esistente e richiede l'ID del file (non della cartella). Confonderle è un errore comune.

5 Concetti chiave per progetti futuri
Pattern: Binary → base64 → Binary Ogni volta che devi inviare un file a un'API REST che non accetta upload multipart, usa questo pattern: Extract from File per convertire in base64, includi la stringa nel JSON body, poi Convert to File per tornare a binario dalla risposta. È il pattern standard per lavorare con file nelle HTTP API.
Accesso a nodi precedenti con $('NomeNodo').item.json In n8n ogni nodo può accedere ai dati di qualsiasi nodo precedente — non solo dell'immediato predecessore. Nel nodo HTTP Request il body usa sia $json.data (dal nodo precedente Extract from File) sia $('On form submission').item.json[...] (dal trigger, due nodi prima). Questa sintassi è fondamentale per costruire payload API complessi.
httpQueryAuth: come funziona l'autenticazione Gemini La Gemini API accetta la chiave come query parameter nell'URL: ?key=API_KEY. In n8n si configura con Authentication: Generic Credential Type → HTTP Query Auth, impostando name: key e value: <la tua API key>. Non usare API Key standard di n8n perché usa header HTTP diversi.
Naming automatico dei file con $now.toFormat() $now è l'oggetto Luxon DateTime disponibile in tutte le espressioni n8n. toFormat('ddMMyyyy_HHmm') produce timestamp leggibili come 28052026_1322. Usa sempre un timestamp nel nome del file su Drive per evitare sovrascritture accidentali.
Create vs Update in Google Drive Create: crea un nuovo file — richiede l'ID della cartella di destinazione.
Update: sovrascrive un file esistente — richiede l'ID del file da aggiornare.
Passare un ID di cartella a Update è l'errore più comune: n8n non genera un errore ma rinomina e sovrascrive la cartella stessa.
